# 新しい市区町村を追加する手順書

本ポータルサイト（オール介護ポータル）に、新しい市区町村（例: 春日市）の事業所データを追加する際の手順書です。

---

## 概要
本システムは、`data/facilities.csv` に登録されている公開データから市区町村のリストを動的に生成します。そのため、基本的には **CSVファイルにデータを追加するだけで、市区町村の検索フィルターやエリア一覧に自動的に反映されます**。

一部、表示調整やニュース連携を行う場合は関連ファイルの編集が必要になります。

---

## 手順 1: CSVデータへの追加（必須）

`data/facilities.csv` を編集し、追加したい市区町村の事業所データを追加します。
※編集の際は、[data/README.md](file:///c:/Projects/care-portal_v2/data/README.md) の記述（Excel等での編集方法、カンマや改行コードの扱い）も合わせてご確認ください。

### 主な列の設定値

CSVの末尾に行を追加し、以下の形式でデータを入力します。

| 列名 (CSVヘッダー) | 設定値の例・注意点 |
| :--- | :--- |
| **facilityId** | 重複しない一意のID（例: 福岡市が `f000001`〜、北九州市が `f001118`〜 なので、`f002000` などの区切りの良い番号、または末尾の連番） |
| **officeNumber** | 10桁の介護保険事業所番号 |
| **companyId** | 運営法人の管理用ID（任意 / 空欄可） |
| **companyName** | 運営法人名（例: `株式会社〇〇ケア`） |
| **name** | 事業所名 |
| **slug** | 詳細ページのURL用ID。英数字とハイフンのみ、他と重複不可（例: `kasuga-care-center`） |
| **prefecture** | 都道府県名（例: `福岡県`） |
| **city** | 追加する市区町村名（例: `春日市`）<br>※**ここに設定した文字列が、フィルターの選択肢として動的に抽出されます。** |
| **ward** | 政令指定都市の区名（例: 福岡市東区なら `東区` / 春日市のように区がない場合は空欄） |
| **area** | 表示用のエリア名。通常は `市区町村名 + 地区名`（例: `春日市日の出町`）<br>※**丁目の表記がある場合は、「春日市小倉四丁目」ではなく「春日市小倉」のように丁目を除去した値に正規化して登録します。**（エリア一覧での重複乱立を防ぐため） |
| **type** | サービス種別。以下のいずれか（完全一致）を指定します：<br>`訪問介護` / `居宅介護支援` / `通所介護` / `訪問看護` / `福祉用具貸与` / `住宅型有料老人ホーム` |
| **icon** | 表示用絵文字（例: `🏠` / `📝` / `🩺` / `☀️` ） |
| **color** | 表示の配色テーマ。以下から選択：<br>`emerald` / `amber` / `sky` / `orange` / `violet` / `rose` など |
| **desc** | 事業所の簡単な紹介テキスト |
| **tags** | タグ。複数ある場合は `|` で区切る（例: `訪問介護｜生活援助｜身体介護`） |
| **address** | 所在地の住所 |
| **tel** | 電話番号（ハイフンあり） |
| **hours** | 営業時間（例: `平日 9:00〜18:00`） |
| **website** | ホームページのURL（任意 / なければ空欄） |
| **serviceArea** | 対応エリア |
| **isPublished** | `true`（サイト上に掲載・表示する場合は必ず `true` にします） |
| **contactStatus** | 接触ステータス（例: `未接触` など管理用の値） |
| **updatedAt** | 更新日（例: `2026-07-01`） |
| **createdAt** | 作成日（例: `2026-07-01`） |

## 手順 2: 各種コンポーネントの表示調整（任意・推奨）

### 2.1. トップページのエリア表示（市区町村単位への変更）
トップページ（[app/page.tsx](file:///c:/Projects/care-portal_v2/app/page.tsx)）の「エリアから探す」は市区町村（`city`）単位で表示されるよう自動化されています。また、表示関数 `displayArea` も自動で市名を取り除く動的処理になっておりますので、個別のコード変更は原則不要です。


### 2.2. 制度・支援情報（ニュース）の地域フィルター追加
もし、春日市独自の補助金や加算などのニュースカードを掲載し、地域別で絞り込めるようにしたい場合は、以下のファイルを編集します。

* **編集ファイル**: [app/care-news/page.tsx](file:///c:/Projects/care-portal_v2/app/care-news/page.tsx)
  1. `regionOptions` 配列に `"春日市"` を追加します。
  2. `getRegionTargets` 関数に `"春日市"` が選択されたときの判定処理を追加します。

```typescript
// 1. regionOptions の更新 (8行目付近)
const regionOptions = [
  "すべて",
  "全国",
  "福岡県",
  "福岡市",
  "北九州市",
  "春日市", // 追加
] as const;

// 2. getRegionTargets の更新 (33行目付近)
function getRegionTargets(region: string) {
  if (region === "すべて") return null;
  if (region === "全国") return ["全国"];
  if (region === "福岡県") return ["全国", "福岡県"];
  if (region === "福岡市") return ["全国", "福岡県", "福岡市"];
  if (region === "北九州市") return ["全国", "福岡県", "北九州市"];
  if (region === "春日市") return ["全国", "福岡県", "春日市"]; // 追加
  return null;
}
```

* **編集ファイル**: [data/care-news-cards.ts](file:///c:/Projects/care-portal_v2/data/care-news-cards.ts)
  * 対象 of the news data の `regions` プロパティに `["春日市"]` または `["福岡県"]` などを設定します。

## 手順 3: ローカル環境での動作確認

変更を保存後、ローカルの開発サーバーで正しく反映されているか確認します。

1. **開発サーバーの起動**
   ```bash
   npm run dev
   ```
2. **ブラウザでの確認**
   * [http://localhost:3000/facilities](http://localhost:3000/facilities) にアクセスします。
   * 市区町村フィルターに「春日市」が動的に追加されていることを確認します。
   * 「春日市」を選択した際に、CSVに追加した事業所情報が正しくリストアップされること、詳細ページへのリンクが正しく動くことを確認します。
3. **ビルドテストの実行** (型エラーやNext.jsのビルドエラーがないか事前にチェック)
   ```bash
   npm run build
   ```

---

## 手順 4: GitHubへのコミット & プッシュ

動作確認が完了したら、Gitを使ってリモートリポジトリへプッシュします。

1. **変更ファイルのステータス確認**
   ```bash
   git status
   ```
   *(facilities.csv や編集したコードファイルが変更対象としてリストされているか確認します)*

2. **変更ファイルのステージング**
   ```bash
   git add data/facilities.csv
   # (他のファイルを編集した場合はそれらも add します。例: git add app/page.tsx)
   ```

3. **コミットの作成**
   分かりやすいコミットメッセージを記述します。
   ```bash
   git commit -m "feat: 春日市の事業所データ追加および手順書の作成"
   ```

4. **GitHubへのプッシュ**
   現在のブランチ（通常は `main`）にプッシュします。
   ```bash
   git push origin main
   ```
   *(※プルリクエストを経由する場合は、対応する作業ブランチにプッシュしてください)*
