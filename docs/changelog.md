### 変更 (Changed)
* **サービス種別フィルターの大分類化 (グループ検索の導入)**:
  - 一覧画面の「サービス種別」フィルターが細分化されすぎていたため、ユーザー向けに8つの大分類（訪問サービス、通所サービス、居宅・相談、入居・施設、短期入所、福祉用具、その他、すべて）にグループ化して表示するように変更しました。
  - `data/facilities.csv` 内の正式なサービス種別名（`type`）および詳細ページ等のバッジ表示は一切変更せず、一覧画面の検索UIおよび絞り込みロジックのみを大分類化しました。
  - 今後新しいサービス種別が追加された場合でも「その他」に自動マッピングされるフォールバック設計を実装。市区町村・エリア・キーワード検索との併用も正常に維持されます。
* **検索UIの「現在地が分かる2階層構造」への改善**:
  - トップページ（例: `/facilities?type=訪問介護`）やURLクエリから直接特定のサービス種別（個別 `type`）が指定されて遷移した際に、所属する大分類（第1階層）を自動判定して「選択中」にしつつ、直下にその大分類に属する個別サービス種別（第2階層）を表示・選択状態にする2階層構造フィルターを実装しました。
  - 選択されている絞り込み条件（市区町村・エリア・大分類・個別タイプ）に応じて、ページ見出し（例: `太宰府市の訪問介護事業所`）を動的に切り替える仕組みを追加しました。
  - 現在の絞り込み状態が視覚的にすぐ分かる、シンプルでクリック可能な動的パンくずリスト（例: `TOP ＞ 介護事業所を探す ＞ 太宰府市 ＞ 訪問サービス ＞ 訪問介護`）を画面上部に追加しました。
* **一覧画面上部のコンパクト化と一覧見出し・件数の動的追加**:
  - PC・スマホ表示において縦方向の不要な余白を縮小し、`h1` の文字サイズ縮小（5xl➔3xl）、検索窓やボタンのパディングの縮小（py-4➔py-3）、説明文フォントサイズ縮小（sm➔xs/sm）により、ページ上部の情報密度を向上しコンパクト化しました。
  - 絞り込みUIの直下に、検索条件と連動した動的な一覧見出し（`h2`：例: `太宰府市の訪問介護事業所一覧`）および件数表示（例: `全 16 件`）を追加しました。
  - 最上部の `h1`（ページ全体の現在地）と一覧見出し `h2`（リストの開始位置）で役割を分けることで重複感を排除し、かつ将来的にページネーション用の表示構造（例: `1〜20件を表示`）へ移行しやすいマークアップ設計を施しました。
  - 旧レイアウトで最上部に直書きされていた `掲載件数：` 表記は、一覧見出しの横へ統合・移動しました。
* **介護事業所一覧の並び順の明示的なソート（並び替え）処理の実装**:
  - CSVの読み込み順（追加時期）で決まっていた不規則な表示順を廃止し、フィルタリングされた配列に対して明示的なソートルールを追加しました。
  - 【市区町村指定なし】: 定義された優先順（`福岡市 ➔ 北九州市 ➔ 春日市 ➔ 大野城市 ➔ 太宰府市`）にソート。
  - 【市区町村指定あり】: 指定市内の区・エリア（例: 福岡市は `東区 ➔ 博多区 ➔ 中央区 ➔ 南区 ➔ 城南区 ➔ 早良区 ➔ 西区` の順、北九州市は `門司区 ➔ 小倉北区...` の順）にソート。区・エリアを持たない市はエリア名の五十音順に並び替えます。
  - 【同一区・エリア内】: `localeCompare("ja")`（日本語の自然比較ロケール）を使用して、事業所名を日本語ロケールに基づく文字列順にソートします。
  - 【将来の拡張性】: ページネーション（1ページ20件等）を今後導入した際にもソートが維持されるよう、ページ分割処理より前に並び替えを完了する設計としました。
  - データ件数はソート前後で変化せず、マスターCSVの行順やデータ自体（`facilityId`, `slug` 等）は一切変更していません。
* **20件／ページのページネーション（ページ分割）機能の追加**:
  - 1ページあたり最大20件のみ表示するスライス処理（`slice`）をソート処理の直後に組み込み、ページの長大化とフッターへの到達困難なUX問題を解消しました。
  - 一覧見出し直下の件数表示をページネーションと連動する表現（例: `全 785 件中 21〜40 件を表示`、1件のみの場合は `全 1 件中 1 件を表示`）に動的拡張しました。
  - 事業所カード一覧の下部に、現在のページ、前後のページ、最初/最後のページ、および省略記号（`...`）を考慮したレスポンシブなページネーションUIを新規実装しました。
  - 【クエリパラメータの維持】: ページを変更しても、`type`、`area`、`city`、`keyword` 等の既存の検索フィルター条件が破棄されないよう `createFacilitiesHref` リンク生成関数を拡張しました。また `page=1` の場合はURLに `page` を付与せず通常URLに整理します。
  - 【フィルター変更時のリセット】: ユーザーが検索条件やサービス種別、市区町村などを変更した際は、必ず自動的に `page=1`（1ページ目）にリセットして表示する仕組みを適用。
  - 【不正なページ値へのフォールバック】: `page=0`, `page=-1` や `page=abc` などの不正値は 1 ページ目に補正、最大ページ数を超える `page=999` の場合は最終ページに自動的に丸め込むフェールセーフ処理を実装しました。
  - TypeScriptの静的ビルドテスト（`npm run build`）も完全に成功することを確認しました。
* **操作時の視覚的フィードバック改善（フェーズ1: 共通クラス定義とHeader/Footerへの適用）**:
  - 操作感の向上とアクセシビリティ対応のため、ポータル全体で統一して使用できるインタラクション用の共通ユーティリティ（`btn-primary`, `btn-secondary`, `link-interactive`）を `app/globals.css` に定義しました。
  - ボタン系（ホバーで明度低下＋影、クリック時に微小な押し込み `active:scale-[0.98]`、フォーカス時に `focus-visible:ring-emerald-600`）およびリンク系（ホバー時にカラーシフト、フォーカス時に `focus-visible:ring-emerald-500`）の共通状態変化を設定。
  - 【アニメーション低減対応】: `prefers-reduced-motion: reduce` メディアクエリを定義し、OS側でアニメーションを制限しているユーザーに対しては transition や scale などの微小変形効果を一括無効化する安全設計を施しました。
  - 【Header/Footerへの適用】: ヘッダーのロゴ・ナビゲーションリンク、およびフッターの各テキストリンク、ネットワーク外部リンクカード（L32〜L87、`active:scale-[0.99]` と `focus-visible:ring-brand-red/60` 等を付与）に対して、レイアウトや余白を崩すことなく上記のクラスおよびフォーカススタイルを適用。キーボード操作時の明確な現在位置明示に対応しました。
  - モバイルメニューボタンの開閉等の既存ロジックは一切変更していません。
* **介護事業所一覧ページの主要操作要素への操作フィードバック追加とフィルター物理的押し込み強化**:
  - 既に `globals.css` で定義した共通インタラクションスタイルとの一貫性を持たせるため、一覧ページ（`app/facilities/page.tsx`）の主要なボタン・リンク（検索ボタン、大分類フィルター、詳細分類フィルター、市区町村フィルター、エリアフィルター、ページ送りコントロール）に視覚フィードバックを追加しました。
  - 【検索ボタン】: プライマリボタンとしての操作感（ホバーで明度低下＋影、クリック時に `active:scale-[0.98]`、フォーカス時にエメラルドリングを表示し、トランジションを `duration-100` へ最適化）を追加。
  - 【各種フィルターボタン（物理的押し込みの強化）】: 既存の選択状態（背景色や文字色）やレイアウトは一切崩さずに、押した瞬間のフィードバックを高める複合スタイル（クリック時に強めの `active:scale-[0.95]`、内側の影 `active:shadow-inner`、明度低下 `active:brightness-90`）を適用。派手すぎず、かつ物理的にボタンが沈み込んだような自然なクリック感を実現しました。
  - 【ページネーション現在ページへの配慮】: 現在選択中であるアクティブなページ番号リンクについては、クリック不可であるため不要な active 効果（スケール縮小等）を付与しないよう制御を組み込みました。
  - 事業所カード、トップページ、詳細ページ、お問い合わせフォーム等の他の要素は一切変更していません。
