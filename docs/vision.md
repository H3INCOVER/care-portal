# プロジェクトビジョンと運営理念 (Vision)

本プロジェクト（オール介護ポータル）の目的、提供価値、および運営とAI活用に関する指針について説明します。

---

## 1. プロジェクトの目的と理念

本プロジェクトの目的は、**「地域における介護サービス情報へのアクセス障壁を極限まで下げること」** です。
介護保険サービスや事業所の情報は公的に開示されていますが、一般のユーザーにとって探しやすく整理された情報ポータルは未だ不足しています。
私たちは、シンプルかつ高速で、直感的に目的の事業所に辿り着けるポータルサイトを提供します。

---

## 2. 提供する価値

1. **直感的なエリア・サービス検索**:  
   ユーザーが迷わないよう、市区町村 ➡ 種別 ➡ 詳細エリアというシンプルな階層化ナビゲーションを提供します。
2. **情報の信頼性**:  
   自治体などが公開する公式のオープンデータをベースにしながら、手動の更新対応を含めて常に最新かつ安全な情報を提示します。
3. **高速なレスポンスと使いやすさ**:  
   Next.jsの静的ジェネレーション（SSG）を最大限に活用し、モバイル環境や回線速度の遅い環境からでも瞬時に読み込める軽量なUXを実現します。

---

## 3. 運営・開発方針

本プロジェクトの開発において最も重視するのは、一時的な機能完成ではなく、**「持続可能な仕組みの構築」** です。
開発プロセスにおいて以下の3つの負債を徹底的に防ぎます。

* **技術負債 (Technical Debt)**:  
  複雑すぎるフレームワークや外部依存、不必要なデータベースサーバーの導入を避け、極限までシンプルなアーキテクチャを維持します。
* **理解負債 (Comprehension Debt)**:  
  「コードそのものが何を目的として書かれているか」を開発者がいつでも説明できるドキュメント（`docs/`）を併せて作成します。
* **認知負債 (Cognitive Debt)**:  
  設計やデータの流れがブラックボックス化するのを防ぎ、新入メンバーや他のAIが参加した瞬間に全体像を把握できる構成を目指します。

---

## 4. AI活用方針 (AI Collaboration Policy)

本プロジェクトでは、人とAIがそれぞれの得意領域を活かした共同開発体制をとります。

* **AIの役割**:  
  情報の調査・収集、最適な実装案の提案、コードの記述、仕様書や運用手順書といったドキュメント作成の補助を担当します。
* **人間の役割**:  
  プロジェクトの根本的な目的の定義、設計判断における選択肢の決定、優先順位の調整、および最終的な品質・動作のチェックを担当します。

AIはただコードを高速に書き出すツールではなく、**「設計と思考、そして運用の仕組みを支援するパートナー」** として位置づけます。
